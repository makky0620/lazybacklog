# Demo Mode Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `--demo` CLI flag that runs lazybacklog with hardcoded English mock data, requiring no config file or API key.

**Architecture:** Detect `--demo` in `main()`, create a synthetic `Config` via `mock::demo_config()`, pass `demo_mode: bool` through `run()` into `AppState`. Each fetch function (`fetch_projects`, `fetch_statuses`, `fetch_issues`) checks `state.demo_mode` and sends mock `AppEvent`s synchronously instead of making HTTP calls.

**Tech Stack:** Rust, ratatui, tokio, crossterm. No new dependencies required.
