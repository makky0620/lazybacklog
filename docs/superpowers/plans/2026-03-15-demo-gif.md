# Demo GIF Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create a reproducible VHS-based GIF demo of lazybacklog and embed it in both READMEs.

**Architecture:** A `.tape` script in `demo/demo.tape` drives VHS to produce `demo/demo.gif`. The GIF is committed to the repo so it renders directly on GitHub. The READMEs get the image inserted above the existing `cargo run -- --demo` block.

**Tech Stack:** VHS (`brew install vhs`), Rust release binary (`cargo build --release`)

---

## Chunk 1: Install VHS and create demo.tape

### Task 1: Install VHS

**Files:** none

- [ ] **Step 1: Install VHS via Homebrew**

```bash
brew install vhs
```

Expected: `vhs` command available.

- [ ] **Step 2: Verify VHS is installed**

```bash
vhs --version
```

Expected: prints version string (e.g. `vhs version 0.x.x`).

### Task 2: Create demo/demo.tape

**Files:**
- Create: `demo/demo.tape`

- [ ] **Step 1: Create the `demo/` directory**

```bash
mkdir -p demo
```

- [ ] **Step 2: Write `demo.tape`**

Create `demo/demo.tape` with the following content.
Note: `Output` and the binary path are both relative to the tape file's location (`demo/`), so `Output demo.gif` produces `demo/demo.gif` and `../target/release/lazybacklog` resolves to the repo root binary.

```
Output demo.gif

Set Width 220
Set Height 50
Set FontSize 14
Set Theme "Dracula"
Set Framerate 24

Type "../target/release/lazybacklog --demo"
Enter
Sleep 1s

# Select first project
Enter
Sleep 500ms

# Browse issues with j (vi-style)
Type j
Sleep 300ms
Type j
Sleep 300ms
Type j
Sleep 300ms

# Open issue detail
Enter
Sleep 500ms

# Close detail
Escape
Sleep 500ms

# Start search (sleep after / to let UI enter search mode)
Type /
Sleep 300ms

# Type query — matches "Fix login page layout on mobile" (DEMO-2)
Type "login"
Sleep 500ms

# Show match highlight
Sleep 1s

# Exit search mode before quitting
Escape
Sleep 300ms

# Quit
Type q
```

- [ ] **Step 3: Commit demo.tape**

```bash
git add demo/demo.tape
git commit -m "demo: add VHS tape script"
```

## Chunk 2: Generate demo.gif

### Task 3: Build release binary and run VHS

**Files:**
- Create: `demo/demo.gif` (generated)

- [ ] **Step 1: Build the release binary**

```bash
cargo build --release
```

Expected: `target/release/lazybacklog` exists.
**The binary must be present before Step 2 — VHS cannot compile Rust and will fail silently if the binary is missing.**

- [ ] **Step 2: Generate the GIF**

Run from the repo root:

```bash
vhs demo/demo.tape
```

Expected: `demo/demo.gif` is created. VHS will open a headless terminal, run the app, and record the keystrokes. This takes ~20 seconds.

- [ ] **Step 3: Verify the GIF looks correct**

Open `demo/demo.gif` in a browser or image viewer. Confirm:
- Project select screen appears
- Issues load and cursor moves down 3 rows
- Issue detail popup opens and closes
- Search bar appears with `login` typed and a match highlighted
- App exits cleanly

- [ ] **Step 4: Commit demo.gif**

```bash
git add demo/demo.gif
git commit -m "demo: add generated demo GIF"
```
